# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in this project, please email security@example.com with:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

Please do not open public GitHub issues for security vulnerabilities.

## Security Practices

This project follows these security practices:

- Dependencies are regularly updated to fix known vulnerabilities
- Environment variables are used for sensitive configuration
- Emails and API endpoints are not hardcoded in code
- All secrets should be managed through environment variables
- Use `.gitignore` to prevent accidental commits of sensitive files
